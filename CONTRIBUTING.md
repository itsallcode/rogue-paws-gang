# Contributing

## Quick Start

1. Fork the repository
2. Create a feature branch: `git checkout -b documentation/<change-title>`
3. Make your changes
4. Validate your changes
5. Submit a pull request

## What We Accept

- Bug fixes
- New game content (characters, scenarios, rules)
- Translation improvements
- Documentation updates
- Balance adjustments

## Guidelines

- Keep the game's tone and style consistent
- Test any rule changes with actual gameplay
- Use clear, descriptive commit messages
- Update relevant documentation

## Questions?

Open an issue or start a discussion.
